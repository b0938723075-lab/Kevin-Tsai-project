# 🧠 Kevin Tsai 專案記憶與執行規範 (Memory)

這份文件記錄了柔柔對於【蔡康永/Kevin Tsai 輿情分析報告】的客製化要求與資料呈現規範。
未來在進行任何系統修改或產出報告時，請嚴格遵守以下守則：

## 📊 報告資料呈現方式 (四大分類與原則)

1. **維持四大分類框架**：
   報告必須固定包含以下四大區塊，且不可使用舊版的「正面/負面聲量」格式：
   - 📱 【社群動態】
   - 📚 【書籍作品】
   - 📺 【主持節目】
   - 📰 【相關新聞】

2. **嚴格排除生平百科 (維基百科等)**：
   在「📚 【書籍作品】」或者是任何分類中，**嚴禁**放進維基百科 (wikipedia.org)、Wikiwand (wikiwand.com)、百度百科這類記載生平的基本資料。報告內容應專注於他實際創作出版的書籍、書評或節目金句。

3. **資訊精準控制 (3~5 則最新消息)**：
   每個分類只需挑選「最新」的 3~5 則重點資訊。
   - 資料過多時：務必過濾雜訊，精選最核心的 3~5 則。
   - 資料不足時：有多少列多少，但千萬不可隨意塞入無關的百科資料充數。

4. **⭐ 時間區間（重大更新 2026-04-03 起生效）**：
   - **觀測區間**：改為僅抓取「當天」的最新資料。不再回溯到 2026 年 2 月。
   - **起始日期**：2026-04-03 開始實施此規範，每日產出一份當天的輿情報告。
   - **空窗期對策**：如果當天搜尋不到足夠的新資料，AI 大腦可以依照近期幾天的公開動態進行補充，但必須明確標注來源，且絕不編造虛構假新聞。
   - **搜尋查詢語法**：使用 `after:YYYY-MM-DD`（設定為前一天的日期），確保只抓到最近 24 小時以內的資料。
   - **報告標頭顯示**：日期顯示為當天日期（例如 `📆 2026-04-03 每日輿情報告`），不再顯示 `2026.02 - 至今`。

5. **歷史紀錄文章選項**：
   報告網頁必須包含一個「📅 歷史紀錄文章」的日期下拉選擇器，讓柔柔可以依照日期選擇瀏覽過往任何一天的報告。選項中需標示星期幾，並在當天的選項旁標註「← 今日」。

6. **一週摘要統整**：
   每份報告的結尾（包含網頁版 **與** LINE/Telegram 推播版）都必須附上「📊 一週摘要統整」區塊，包含：
   - 📌 本週平均情緒分數
   - 📈 情緒趨勢方向（上升 / 持平 / 下降）
   - 📄 本週報告天數
   - 📆 每日情緒分數走勢圖（網頁版限定）

7. **手機推播內容與網頁版必須完全同步**：
   - Pipeline 流程中，在「產出網頁」完成後、「發送通知」之前，必須先「上傳 Git Push 到 GitHub」，確保手機點開連結時看到的網頁是最新的。
   - 手機收到的四大分類文字報告，必須與網頁上的分析卡片內容完全一致。

## ⚠️ 開發作業注意事項
- 每次修改程式碼前，請務必先回顧此 `memory.md`，確保新的改動不會破壞上述的報告呈現邏輯。
- 所有推播模組（如 LINE、Telegram）與網頁介面，都必須同步對齊此 3~5 則與四大分類的排版規則。
- Pipeline 執行順序：搜集 → 分析 → 產出網頁 → **上傳 GitHub** → 發送通知。
